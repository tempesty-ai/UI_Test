# QA Visual Tester

Microsoft Playwright MCP를 활용하여 실제 브라우저로 웹 애플리케이션을 시각적으로 QA 검증하고, 이슈를 체계적으로 기록하는 스킬이다.

## 전체 흐름

**단일 타겟**: 1. 요구사항 수집 → 2. 테스트 시트 작성 → 3. Playwright 실행 → 4. 이슈 기록

**다중 타겟** (targets.json에 2개 이상): 각 타겟에 대해 2→3→4단계를 순서대로 반복 실행 → 전체 멀티 요약 출력

---

## 1단계: 요구사항 수집

### targets.json가 있는 경우 (우선 확인)

`targets.json` 파일이 존재하면 파일을 읽어서 모든 타겟 목록을 확인한다.

```json
{
  "targets": [
    { "url": "...", "name": "...", "focus": ["..."] },
    { "url": "...", "name": "...", "focus": ["..."] }
  ]
}
```

- **타겟이 1개**면 해당 URL로 단일 테스트 진행
- **타겟이 2개 이상**이면 다중 타겟 모드로 진입 — 각 타겟을 순서대로 테스트

### 직접 URL을 받은 경우

사용자로부터 다음 정보를 확인하라 (없으면 질문):

- **대상 URL** — 테스트할 페이지 주소
- **테스트 범위** — 어떤 기능/화면을 검증하는가 (없으면 전반적인 UI/UX 기준으로 진행)
- **중점 항목** — 특별히 확인해야 할 사항
- **환경 정보** — 계정/인증이 필요한가

정보가 충분하면 바로 2단계로 넘어간다.

---

## [다중 타겟 모드] 순차 실행 규칙

타겟이 2개 이상일 때 아래 규칙을 따른다.

1. **시작 전 전체 타겟 목록을 출력**한다:
   ```
   🎯 다중 타겟 QA 시작 — 총 N개 사이트
   [1] https://site-a.com
   [2] https://site-b.com
   ...
   ```

2. **각 타겟마다 2→3→4단계를 완전히 완료**한 뒤 다음 타겟으로 넘어간다.
   - 타겟별 디렉토리를 독립적으로 생성: `[YYYYMMDD]_[name]/`
   - 타겟 사이에 구분선 출력: `━━━ [2/N] 다음 타겟 시작: https://... ━━━`

3. **한 타겟에서 P1 Critical 이슈 발견 시**: 해당 타겟 테스트를 마치고 다음 타겟으로 계속 진행한다. (중단하지 않음)

4. **모든 타겟 완료 후 멀티 요약**을 출력한다 (아래 참고).

---

## 2단계: QA 테스트 시트 작성

테스트 실행 전에 반드시 구조화된 테스트 시트를 먼저 작성하고 사용자에게 보여준다.

### 테스트 시트 형식

```
## QA 테스트 시트 — [대상 URL]
생성일시: YYYY-MM-DD HH:MM
테스트 환경: Chrome (Playwright)

| # | 카테고리 | 테스트 항목 | 검증 방법 | 예상 결과 | 우선순위 |
|---|---------|-----------|---------|---------|---------|
| 1 | 페이지 로딩 | 메인 페이지 정상 로드 | URL 접속 후 핵심 요소 확인 | 200 응답, 주요 콘텐츠 표시 | P1 |
| 2 | 레이아웃 | 헤더/푸터 렌더링 | 시각적 확인 | 깨짐 없이 표시 | P1 |
...
```

### 카테고리 기본 목록 (요구사항에 따라 조정)

- **페이지 로딩** — 접속 속도, HTTP 상태, 리소스 로딩
- **레이아웃/UI** — 깨짐, 겹침, 정렬, 폰트 렌더링
- **네비게이션** — 메뉴, 링크, 브레드크럼, 뒤로가기
- **폼/인터랙션** — 입력 필드, 버튼 동작, 유효성 검사
- **반응형** — 데스크톱(1280px), 태블릿(768px), 모바일(375px)
- **콘텐츠** — 텍스트 깨짐, 이미지 로딩, 빈 상태
- **에러 처리** — 잘못된 입력, 404, 네트워크 에러 메시지

우선순위: P1(크리티컬) > P2(주요) > P3(마이너)

---

## 3단계: Playwright MCP로 테스트 실행

**중요: `playwright` MCP 서버의 도구들을 사용하여 실제 브라우저를 열고 직접 검증하라.**

### Playwright MCP 사용 패턴

```
# 브라우저 시작 & 페이지 이동
playwright_navigate → URL로 이동
playwright_screenshot → 현재 화면 캡처
playwright_click → 요소 클릭
playwright_fill → 폼 입력
playwright_select_option → 드롭다운 선택
playwright_evaluate → JavaScript 실행
playwright_resize → 뷰포트 크기 변경 (반응형 테스트)
```

### 각 테스트 항목 실행 방식

1. 해당 테스트에 맞게 브라우저 조작
2. `playwright_screenshot`으로 화면 캡처
3. 결과 판정: PASS / FAIL / WARNING
4. FAIL/WARNING이면 즉시 이슈 기록 (4단계)

### 반응형 테스트 뷰포트

- 데스크톱: 1280×800
- 태블릿: 768×1024
- 모바일: 375×812

## 4단계: 이슈 기록

### 디렉토리 구조

```
D:\code\qa\
└── [YYYYMMDD_HHMMSS]_[도메인]/      ← 테스트 세션 루트
    ├── QA_SHEET.md                   ← 전체 테스트 시트 + 결과 요약
    ├── SUMMARY.md                    ← 이슈 요약 리포트
    └── issues/
        ├── ISSUE-001_[제목]/
        │   ├── ISSUE.md              ← 이슈 상세 기록
        │   └── screenshot.png        ← 캡처 이미지
        ├── ISSUE-002_[제목]/
        ...
```

### ISSUE.md 형식

```markdown
# ISSUE-[번호]: [이슈 제목]

- **심각도**: P1 Critical / P2 Major / P3 Minor
- **카테고리**: 레이아웃 / 기능 / 콘텐츠 / 성능 등
- **발견 일시**: YYYY-MM-DD HH:MM
- **URL**: 이슈 발생 URL
- **재현 조건**: 뷰포트, 브라우저, 선행 동작

## 현상
[실제 관찰된 내용]

## 예상 동작
[정상적으로 기대되는 동작]

## 재현 방법
1. ...
2. ...

## 스크린샷
![screenshot](./screenshot.png)

## 비고
[추가 컨텍스트, 관련 이슈 등]
```

### QA_SHEET.md 결과 업데이트 형식

```
| # | 테스트 항목 | 결과 | 이슈 링크 | 비고 |
|---|-----------|------|---------|-----|
| 1 | 메인 페이지 로드 | ✅ PASS | - | - |
| 2 | 헤더 렌더링 | ❌ FAIL | ISSUE-001 | 모바일서 깨짐 |
```

### SUMMARY.md 형식

```markdown
# QA 테스트 결과 요약

- **테스트 일시**: YYYY-MM-DD HH:MM
- **대상 URL**: https://...
- **총 테스트 항목**: N개
- **PASS**: N개 | **FAIL**: N개 | **WARNING**: N개

## 발견된 이슈 (심각도 순)

### P1 Critical
- [ISSUE-001](./issues/ISSUE-001_제목/ISSUE.md): 이슈 요약

### P2 Major
...

## 권고 사항
[종합 의견 및 우선 수정 권고]
```

---

## 진행 커뮤니케이션

- 테스트 시트 작성 후 "테스트를 시작할까요?" 확인 없이 바로 실행해도 된다 (사용자가 이미 요청했으므로)
- 각 테스트 항목 실행 시 `[진행중] #N: [항목명]` 형태로 진행 상황을 출력하라
- FAIL 발견 시 즉시 알리고 계속 진행하라
- 전체 완료 후 SUMMARY.md 내용을 대화창에 요약해서 보여줘라
- 저장된 디렉토리 경로를 명확히 알려줘라

## 다중 타겟 완료 시 멀티 요약 출력

```

📋 전체 QA 완료 — N개 사이트 테스트 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[1] practicesoftwaretesting.com
    ✅ PASS 6  ❌ FAIL 2  ⚠️ WARN 2
    주요 이슈: 모바일 이미지 미렌더링 (P2)

[2] opencart_demo
    ✅ PASS 8  ❌ FAIL 1  ⚠️ WARN 0
    주요 이슈: 없음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
합계: PASS 14  FAIL 3  WARN 2
긴급 수정 필요 (P1/P2): 3건
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

단일 타겟인 경우 이 요약은 생략하고 SUMMARY.md 내용만 보여준다.
